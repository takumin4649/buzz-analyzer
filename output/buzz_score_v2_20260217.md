## 3. 新スコア（v2）の設計

### 3.1 v1 → v2 変更サマリ

| 要素 | v1 | v2 | 変更理由 |
|------|-----|-----|---------|
| 冒頭パターン | 20点 | **25点** | 数字提示の影響力が最大。配点増 |
| 文字数 | 15点（100-300字） | **20点（0-170字）** | 短い投稿が圧倒的にバズる |
| カテゴリ | 15点（実績報告最高） | **15点（体験談最高）** | データ駆動で順位逆転 |
| 具体的数字 | なし | **15点（新規）** | 唯一の正の相関(+0.225) |
| 簡潔さ | なし | **10点（新規）** | 改行少ない方がバズる(-0.177) |
| 絵文字 | 10点（1-3個最高） | **10点（0個最高）** | 絵文字なしが最もバズる |
| ストーリー性 | 10点 | **5点** | 弱い正の相関、配点縮小 |
| 感情トリガー | 10点 | **削除** | 感情多いほどバズらない |
| CTA | 10点 | **削除** | CTAなしの方がバズる |
| 読みやすさ | 10点 | **→簡潔さに統合** | 箇条書き・改行多い=逆効果 |
| **合計** | **100点** | **100点** | |

### 3.2 v2スコア詳細ロジック

```
1. 冒頭パターン (25点)
   数字提示=25 / 共感=18 / 疑問形=12 / その他=10 / 断定形=8 / 煽り=6 / 呼びかけ=5

2. 文字数 (20点)
   0-80字=20 / 131-170字=17 / 81-130字=13 / 171-220字=10 / 221-300字=4 / 301字+=2

3. カテゴリ (15点)
   体験談系=15 / 問題提起系=15 / ツール紹介系=10 / ノウハウ系=8 / 実績報告系=7 / その他=5

4. 具体的数字 (15点) ★新規
   数字+単位あり=10 / 金額表現あり=+5 / なし=0

5. 簡潔さ (10点) ★新規
   改行0-3=10 / 4-7=7 / 8-12=4 / 13+=1

6. 絵文字 (10点) ★逆転
   0個=10 / 1-2個=6 / 3個+=2

7. ストーリー性 (5点)
   あり=5 / なし=0
```

## 8. v2でも捉えきれない要因と今後の改善案

v2はテキスト特徴量のみでスコアリングしているため、以下の要因は未反映:

| 要因 | 影響度（推定） | 対応方法 |
|------|-------------|---------|
| 投稿者のフォロワー数 | 極めて高い | フォロワー数データを再取得してv3に組み込む |
| 投稿タイミング | 中程度 | 時間帯データは取得済み、ただし今回のデータでは18-21時が低い（n=5で信頼性低） |
| トレンド・話題性 | 高い | 「Claude Code」「Antigravity」等のバズワードは時期依存 |
| リプライ・引用による拡散 | 中程度 | インフルエンサーのリプライやQTによるブースト効果 |
| アカウントの権威性 | 高い | フォロワー数だけでなく、過去のバズ実績も影響 |

**次のステップ:**
1. `buzz_analyzer.py` でフォロワー数を正しく取得して再スクレイピング
2. フォロワー数を組み込んだv3スコアを作成
3. 1-2週間後に自分の投稿の実績データで効果検証
